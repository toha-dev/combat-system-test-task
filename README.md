Геймплей:
Игрок - квадрат (управление WASD). Враги - треугольники. Когда игрок заходит в зону (черный прямоугольник), сверху появляются 3 врага. Враги имеют 1 примитивную атаку (удар в ближнем бою с кулдауном 1.5 сек.).
Игрок имеет 2 вида атаки: базовая и специальная. Базовая (на пробел) - ближний бой, задевает за удар до 2-х врагов. Специальная (правая кнопка мыши) - атака на дистанции (вокруг курсора в радиусе 4 юнитов все враги получают урон).
Игрок и враги имеют способности (Abilities). Игрок имеет сразу все 3 абилки - критический урон (20% на урон х2), уворот (10% на уклонение от атаки противника), щит (10% защита на весь входящий урон).
Враги имеют разные абилки (я там рандомно настроил, 1 из врагов имеет сразу 3)
После смерти игрока - поражение. После убийства всех врагов - победа.

Реализация:
1. Игрок имеет гибкую возможность настройки (можно добавить какие угодно абилки + оружие, но настройка только в эдиторе). Все абилки и виды атаки - ScriptableObject, то есть имеют свои конфиги с настройкой урона, кулдауна, дальности атаки, названия и описания для UI. На один тип атаки можно создать много конфигов (условно ближний бой уровень 1, ближний бой уровень 2..) с разными настройками. Аналогично и для абилки (условно щит уровень 1, щит уровень 2..) с разной защитой в процентах.

2. Враги - имеют только 1 вид атаки (ближний бой), но имеют те же абилити, что и игрок. Можно для каждого врага настроить абилки (но только в эдиторе) на спавнере врагов (EnemySpawner) в сцене Game.

3. Combat System - весь урон проходит через систему, которая учитывает все абилки (того, кто наносит урон и того, кто получает урон). Если атакующий выбивает критический урон (х2 урона), а у получающего урон срабатывает абилка уклонение от атаки, то атака не наносит урона.

4. UI - на HUD экране есть тултипы для абилок (при наведении мыши показывается название + описание). Под абилками 2 белых квадрата (левый - базовая атака игрока, правый - специальная атака), они белые, когда кулдауна нет и удар доступен, красные - когда на них кулдаун. При победе/поражении игра ставится на паузу. Также игра ставится на паузу во время экрана паузы (Escape).

5. Ошибка загрузки в 50% случае - обратите внимание, что ошибка срабатывает после того, как сцена Game загрузилась (это можно увидеть в эдиторе). После срабатывания ошибки (имитация отсутсвия интернет соединения), игрока принудительно выкидывает в главное меню с корректной выгрузкой сцены Game.

Для UI была выбрана MVVM архитектура. Реализация UI максимально простая (На ProjectContext лежат сразу все экраны, через код включается 1 активный экран. Соответствие между гейм объектом и типом экрана устанавливается через enum).
В реальном проекте я бы вынес все экраны в префабы, добавил их в категорию Addressables и асинхронно бы подгружал/выгружал, используя Asset Reference. + прогрев экранов, которые игрок может вызвать в текущий момент времени для уменьшения задержки их показа. + возможность показа несколько экранов одновременно (оверлеи, диалоговые окна) и много всего еще.
